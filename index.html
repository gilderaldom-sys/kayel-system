Auto-save email attachments into the matching project using Power Automate.

Endpoint

SUPABASE_URL_PLACEHOLDER/functions/v1/outlook-attachment

What it does

Sends it the project name and the attachment; it finds (or creates) the matching project, uploads the file to storage, and logs it in the register tagged "Outlook".

Set up in Power Automate

  1. Create a flow: trigger "When a new email arrives" (Outlook 365), filtered to a rule like subject contains a project name, or a specific folder you file deal emails into.
  2. Add "Apply to each" over Attachments.
  3. Add an HTTP action: Method POST, URI as above.
  4. Headers: Authorization: Bearer YOUR_ANON_KEY, Content-Type: application/json.
  5. Body:
{
  "project_name": "Yilu Energy",
  "file_name": "@{item()?['Name']}",
  "content_type": "@{item()?['ContentType']}",
  "file_base64": "@{item()?['ContentBytes']}"
}
Tip: to route each email to the right project automatically, use a Switch on subject/sender before the HTTP call, one branch per project — or hardcode a flow per project inbox rule.

Send the actual file attached (not just a link)

This needs a one-time Azure app registration so Microsoft lets this page send mail as you. It's free and takes a few minutes.
  1. Go to portal.azure.com → Microsoft Entra ID → App registrations → New registration.
  2. Name it anything. Under "Redirect URI" choose platform Single-page application (SPA) and enter this page's exact URL:
  3. After creation, copy the Application (client) ID and Directory (tenant) ID into the boxes below.
  4. In the app's "API permissions" tab, add Microsoft Graph → Delegated → Mail.Send, then grant/accept consent.
